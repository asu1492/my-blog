
Horizontal Scaling (scaling out)
1. What is HS 
	1. Refers to distributing load across multiple devices, generally by adding more machines(nodes or servers). 
	2. E.g. Adding more severs to load balancer 
2. Why do we need HS 
	1. E-Commerce where traffic fluctuates 
3. Challenges Associated with HS 
	1. Setup and Management (e.g. data consistency, network connection b/w servers)
	2. Requires Load Balancing 
4. Benefits of HS
	1. Fault Tolerance 
	2. Redundancy 
	3. Near infinite scalability 

Vertical Scaling (scaling up)
1. What is VS
	1. Increasing capacity of a single machine by adding more memory, CPU or storage. 
2. Why do we need VS
	1. Small-Medium Application where predictable traffic growth. 
3. Challenges Associated with VS 
	1. Single Point of Failure (Do not have redundancy  and fault tolerance)
	2. Limit with how much we can scale a single machine
4. Benefits of VS
	1. Simplicity
