- What is encapsulation in Java?
	- Encapsulation refers to protecting data(_internal state_ of an object) / state and functionality through access modifiers. 
	- It enables exposing state and functionality that is necessary while preventing misuse though excessive exposure of data and functionality. Thus engenders information hiding. 
	- Bundling: Encapsulation is also about bundling the data and the methods that operate on that data within a single unit (class). Thus ensuring data integrity and maintainability. 
	- Also, hides internal implementation details of a class. 
- How is encapsulation implemented in Java?
	- Access Modifier, private fields/instance variable with public getter/setter, immutable objects, package systems, inner classes. 
	- It is implemented through access modifier. Access modifier such as private, protected is appended to fields and methods to prevent accessing fields/state of the object and methods from outside the class their reside. 
	- private/protected fields and methods can be accessed through public methods within the class. 
	- private fields are accessed through public getter and setter methods. 
	- Packages also plays are role in encapsulation. By default fields/method without access modifier cannot be accessed from outside the package. 
- What are the benefits of encapsulation in Java?
	- Data Integrity 
	- Prevent Security related issue and engender security within design. 
	- Aids in abstraction and thus hiding complexity. 
- What is the difference between encapsulation and abstraction?
	- Abstraction works at both design and implementation level while encapsulation is carried out at implementation level. Defining the what (the interface or contract), while implementation details (the how) are handled by encapsulation.
	- Abstraction is achieved through abstract classes, interfaces while access modifiers helps in encapsulation.
	- Purpose, Level of Operation, Implementation Mechanism, Focus, Relationship
- How do access modifiers relate to encapsulation?
	- Access modifiers are used to implement encapsulation in JAVA
	- default(only accessible within the package), public, protected(access within the sub class in different package and in the package), private(access only within the Class). 
- What are getter and setter methods? How do they support encapsulation?
	- These methods are used to access and modify private field/instance variable (state of the object) of the class from outside the class. For. e.g. if a class extended Parent Class. Sub class can access fields of parent class through public setters and getters of the parent class. 
	 %%   A key advantage of using getters and setters is the ability to add validation logic within the setter method. This prevents invalid data from being assigned to the private instance variable. %%
	- Setter and Getter enable data integrity and provide restricted access to field variable of a class. 
	- The concept of read-only or write-only properties using only getters or only setters
- Can encapsulation be broken in Java?
	- Encapsulation can be circumvented through Reflection API. Reflection Object with method [setAccessible(true)](https://bitbucket.org/manhattanassociates/component-dcorder/src/1d10e3a44a60019cf2f95d8054fcf6198802f48e/src/component-test/java/com/manh/cp/dcorder/utility/DcorderContextUtil.java) allows to override access modifier. 
	- Also, using package private (that is not adding any access modifier to any fields or methods). This weakens encapsulation. 
	- **Best Practices:** #best_practices
		- use private keyword liberally
		- have well defined getters, setters
		- Prefer [[Composition]] over [[Inheritance]] 
		- Immutable objects wherever possible
- How does encapsulation support the principle of data hiding?
	- Encapsulation supports data hiding by providing mechanisms to control access to internal data and implementation details. 
	- Through use of access modifiers such as private, protected and package-private (default).
	- Private Fields are hidden and accessed through public getters and setters. 
	- Setter methods can include validation logic to ensure data integrity. Also, getter/setter enable read-only/write-only access. 
- What is the difference between encapsulation and information hiding?
	- information hiding is a _goal_, and encapsulation is a _mechanism_ used to achieve that goal.
	- Encapsulation is a a _technique_ or _mechanism_ used to implement information hiding. It bundles data (fields) and the methods (functions) that operate on that data within a single unit (a class).
