1. Object vs Class 
	1. Class is blueprint of object. Class facilitates how object is to be created. 
2. Superclass and subclass 
3. Superclass and subclass is linked through a relationship called inheritance. 
4. Subclass automatically inherit functions from superclass. 
5. Overriding: When a subclass redefines one of its inherited method. 
6. When designing a class, think about objects that will be created from the class and what the object knows(instance variable/state/the data) and does(method/behavior).
