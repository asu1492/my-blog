Data Structures · First Principles Codex

Where Memory
Becomes Meaning

Every data structure is an answer to one question: given how memory works, how do we arrange information so that the operations we need most are fast? This codex traces each structure from its physical first principle to its derived patterns — the reusable algorithmic moves that follow naturally from the structure's geometry.

"Just as the world is increasingly interpreted in terms of data and information flows, organisms are algorithms and life is data processing." — Yuval Noah Harari, Homo Deus

The atom of all computation

02

String

Language compressed into indices
First Principle
A string is an array of characters with a fixed encoding (ASCII / UTF-8 / UTF-16). Its immutability in most languages is a contract: the length and characters are sealed at creation. All string operations produce new strings by allocating fresh memory. Concatenation is O(n+m) because it copies both. Pattern matching is the canonical string problem — comparing two arrays simultaneously.
Immutable in Java, Python, Go, Rust (by default) Character encoding is an invariant — ASCII is 1 byte, UTF-8 is variable Length is O(1) if stored, O(n) if computed (C-style null-terminated)

    Two-pointer palindrome / reverse checkl=0, r=n-1 converging. At each step, characters must match. O(n) time, O(1) space. Extends to "valid palindrome ignoring non-alphanumeric."
  • →
    Sliding window for longest substringFrequency map in window. Advance right freely, shrink left when constraint violated. Classic: longest substring without repeating chars, min window substring.
  • →
    HashMap for anagram / frequency matchingCount chars in pattern, slide a window over string, compare frequency maps. O(n) with O(26) space for lowercase ASCII.

Linked List

Freedom from contiguity, paid in pointers
First Principle
When elements cannot (or should not) live in contiguous memory, we use pointers to create logical adjacency. Each node stores its value and a reference to the next node. The price: no index arithmetic, so access is O(n). The gain: O(1) insert/delete at any point once you hold a pointer to it — no shifting.
Reachability only via pointer chain Head pointer is the "entry point" — lose it, lose the list null terminates a singly linked list Doubly-linked: prev pointer trades space for O(1) backward traversal

Stack & Queue

Order as a constraint, not just a property
First Principle
Stack (LIFO) and Queue (FIFO) are not new structures — they are arrays or linked lists with restricted access. The restriction is the design: you are forcing a specific ordering discipline onto insertions and removals. This discipline is powerful because many real problems (function calls, task scheduling, BFS) naturally obey LIFO or FIFO order.
Stack: last in, first out — access only the top Queue: first in, first out — insert at back, remove from front Monotonic stack maintains sorted order in the stack at all times
OperationStackQueueWhy
Push / EnqueueO(1)O(1)Append to end
Pop / DequeueO(1)O(1)Remove from defined end
Peek / FrontO(1)O(1)Read without removing
SearchO(n)O(n)No random access
SpaceO(n)O(n)Holds n elements
  • →
    Monotonic Stack (next greater element)Process elements left→right. Maintain stack that is always increasing (or decreasing). When a new element pops something, that's the answer for the popped element. Single pass O(n). Used in: stock span, largest rectangle in histogram, daily temperatures.
  • →
    Bracket / Expression ParsingPush opening brackets. When closing bracket arrives, pop and verify match. The LIFO invariant perfectly mirrors the nesting structure of expressions.
  • →
    BFS with QueueQueue enforces level-by-level exploration. Each node's neighbors are enqueued; nodes dequeued in the order they were discovered. Guarantees shortest path in unweighted graphs.
  • →
    Min-Stack (stack with O(1) minimum)Maintain a second "min-stack" that pushes a new min only when the incoming element ≤ current min. Pop from both in sync. Interview classic that tests invariant thinking.
📞
Call Stack
CPU maintains a call stack. Each function call pushes a frame; return pops it. Stack overflow = exhausted stack space.
🖨️
Print Queue
Print jobs are queued. FIFO ensures fairness. Priority queue adds urgency ordering.
🚦
BFS / Web Crawlers
Web crawlers use a queue of URLs to visit. BFS discovers all reachable pages layer by layer.

HashMap / HashSet

The miracle of amortised O(1)
First Principle
If we want O(1) access by key, we need key → index arithmetic like an array. A hash function converts any key into an integer, then we take it modulo the array size to get a bucket index. Collisions (two keys → same bucket) are resolved by chaining (linked list in bucket) or open addressing. With a good hash and load factor ≤ 0.75, average chain length ≈ 1.
Hash function must be deterministic and uniform Load factor = n/capacity; resize (rehash) when it exceeds threshold Average O(1) all ops; worst case O(n) on hash collision Iteration order NOT guaranteed (unless LinkedHashMap)

    Frequency countingBuild freq[element] = count. Then query in O(1). Foundation for: majority element, top-K frequent, anagram detection, histogram equalization.
  • →
    Complement / existence lookup (Two Sum)For each element x, check if (target - x) exists in the map before inserting x. Single pass O(n). Generalises to any problem where you need "have I seen a related value before?"
  • →
    Caching / MemoizationMap function arguments → computed result. First call computes and stores; subsequent calls retrieve in O(1). Transforms exponential recursion into linear DP.
  • →
    Graph adjacency representationMap> for adjacency list. Faster than 2D matrix for sparse graphs. Also used to map parent → children in tree problems.

Maps domain → IP. Lookup in O(1). TTL-based expiry with LRU eviction when cache is full.

Heap / Priority Queue

Always know the minimum (or maximum)
First Principle
A heap is a complete binary tree stored in an array where the parent is always ≤ (min-heap) or ≥ (max-heap) its children. Storing a tree in an array is possible because completeness means no gaps: node at index i has children at 2i+1 and 2i+2, parent at ⌊(i-1)/2⌋. Insert (bubble up) and extract (bubble down) both take O(log n) because the tree height is ⌊log₂ n⌋.
Heap property: every parent ≤ children (min) or ≥ children (max) Complete binary tree: all levels full except last, filled left to right Root is always the global min/max — O(1) peek
Min-heap stored as array: parent(i)=⌊(i-1)/2⌋, children(i)=2i+1, 2i+2

    Top-K elementsMaintain a min-heap of size K. For each new element, if it's greater than the heap's min (root), pop and push. After all elements, heap contains K largest. O(n log K).
  • →
    Running median (two-heap trick)Max-heap for the lower half, min-heap for the upper half. Median is the root of the larger heap (or average of both roots). Rebalance after each insert. O(log n) per insert, O(1) median query.
  • →
    Dijkstra's shortest pathMin-heap of (distance, node). Extract minimum distance node, relax its neighbors, push updated distances. The heap ensures we always process the globally closest unvisited node. O((V+E) log V).
  • →
    K-way mergePut the first element of each of K sorted lists into a min-heap. Extract min, output it, push the next element from that list. O(N log K) total where N = total elements.
🚑
Hospital Triage
Priority queue orders patients by severity. O(log n) to admit/discharge while always treating the most critical.
📡
Network Routing
Dijkstra with a min-heap powers OSPF routing — routers compute shortest paths across the internet.
⏰
Task Scheduling
OS schedulers use priority queues. CPU-bound vs I/O-bound tasks get different priorities. Next task = O(1) peek.

Tree & BST

Hierarchy without cycles
First Principle
A tree is a connected, acyclic graph. Exactly N-1 edges for N nodes. The root has no parent; every other node has exactly one parent. Trees model hierarchical relationships naturally. A Binary Search Tree adds the ordering invariant: left subtree values < node < right subtree values. This invariant makes every comparison a decision that halves the remaining search space — O(log n) average.
No cycles — unique path between any two nodes BST ordering: left < node < right — must hold at every subtree In-order traversal of BST yields sorted sequence Height-balanced (AVL/Red-Black) trees guarantee O(log n) worst case

  • →
    BFS / Level-order traversalQueue-based, visits nodes level by level. Used for: level averages, zigzag traversal, connecting nodes at same level, minimum depth (first leaf encountered = shortest).
  • →
    Lowest Common Ancestor (LCA)First node in DFS where left subtree contains one target and right contains the other (or node itself is a target). O(n) naive, O(log n) with binary lifting for repeated queries.

    Tree DP (diameter, max path sum)Define f(node) = optimal value of subtree rooted at node. Compute bottom-up in post-order. At each node, combine left and right children results. Diameter = max over all nodes of (left height + right height).

Directories are tree nodes. Files are leaves. DFS powers find; BFS powers ls -R level by level.
🗄️

B-Trees (generalized BST) index database rows. Every SQL range query walks a B-Tree. O(log n) per lookup, disk-optimized branching factor.
🤖
Decision Trees / AI
Random forests and gradient boosted trees are ensembles of binary decision trees. Classification = tree traversal.

Trie

A tree where paths are words
First Principle
In a Trie (retrieval tree), edges represent characters and paths from root to nodes represent prefixes. Storing many strings with common prefixes shares the prefix's memory — "cat", "car", "card" all share the path c→a. Search time is O(L) where L is the length of the key — completely independent of how many strings are stored. This makes Trie asymptotically superior to HashMap for prefix operations.
Each node has up to |alphabet| children (26 for lowercase English) Nodes marked isEnd=true represent complete words All strings with a common prefix share nodes — O(L) search regardless of n
OperationTimeSpaceWhy
Insert word of length LO(L)O(L·|Σ|)Create L nodes, each with |Σ| children slots
Search word of length LO(L)O(1)Follow L edges, check isEnd
Starts-with prefix PO(|P|)O(1)Follow |P| edges, no isEnd check
Delete wordO(L)O(1)Unmark isEnd, prune dangling nodes
Autocomplete (all words with prefix)O(|P| + output)O(output)Navigate to prefix node, DFS from there
Space (n words, avg length L)—O(n·L·|Σ|)Worst case: no shared prefixes

    Autocomplete / TypeaheadNavigate to the prefix node, then DFS/BFS to collect all descendant words. Add word frequency to nodes to return top-K completions efficiently.
  • →
    Word search on board (Trie + DFS)Build a trie of all target words. DFS on the grid, traversing the trie simultaneously. If trie node has no child for current char, prune. Eliminates redundant searches for shared prefixes.
  • →
    Maximum XOR pair (Bit Trie)Store integers as 32-bit binary strings in a trie. For each number, greedily traverse to find the number whose bits are most "opposite." O(32) per query = O(1) per query for 32-bit integers.

Google's suggestion box. Trie of all indexed queries, navigated character by character as you type.
📖
Spell Checker
Dictionary stored as trie. Levenshtein distance computed via BFS on trie. Suggestions = nearest words.
🌐
IP Routing (Radix Trie)
Network prefixes stored in a compressed trie. Longest-prefix matching finds the correct route for any packet.

Segment & Fenwick Tree

Answering range questions in logarithmic time
First Principle
A Segment Tree precomputes aggregates (sum, min, max) for every possible contiguous sub-array — but cleverly, using only O(n) space by storing them in a complete binary tree where each node covers a range. Any query range [l, r] decomposes into O(log n) pre-stored segments. A Fenwick Tree (Binary Indexed Tree) is a more compact, bit-trick-based structure for the specific case of prefix sums — O(log n) update and query with a fraction of the code.
Segment tree node covers range [l,r]; stores aggregate of that range Fenwick tree uses i & (-i) bit trick to define parent/child relationships Lazy propagation defers range updates until child is queried

    Range sum / min / max queries with updatesThe canonical use. Given array with point updates, answer arbitrary [l,r] aggregate queries. Fenwick suffices for sum; Segment Tree needed for min/max.
  • →
    Count inversions (Fenwick + coordinate compression)Compress values to indices. Process right-to-left: for each element, count elements already seen that are smaller using Fenwick prefix sum. O(n log n) — same complexity as merge sort inversion count but more flexible.
📊
Database Analytics
Range aggregations (SUM, MIN, MAX over date ranges) on time-series. Segment trees power OLAP cubes.
🎮
Game Leaderboards
Fenwick tree maintains score-sorted leaderboard. Rank queries and updates both O(log n).
📡
Signal Processing
Windowed statistics over streaming data. Fenwick trees power sliding-window algorithms in real-time analytics.

Graph

The universal structure — everything is a graph
First Principle
All other data structures are special cases of graphs. A tree is a connected acyclic graph. A linked list is a path graph. An array is a path graph with direct address arithmetic. Graphs model any pairwise relationship: V vertices (entities) and E edges (relationships). The two representations are an adjacency matrix (O(V²) space, O(1) edge lookup) and an adjacency list (O(V+E) space, efficient for sparse graphs).
Undirected: edges are symmetric. Directed: edges have direction Weighted: edges carry a cost. Unweighted: all edges equal Connected: path exists between every pair. Disconnected: isolated components DAG (Directed Acyclic Graph): directed, no cycles — models dependencies

    BFS for shortest path (unweighted)Queue-based layer-by-layer expansion. The first time you reach a node, that path is the shortest. Extends to multi-source BFS (start with all sources in queue simultaneously) for problems like "distance to nearest gate."
  • →
    Union-Find for dynamic connectivityTwo operations: union(a,b) merges components; find(a) returns root. With path compression + union-by-rank, both are nearly O(1). Powers: Kruskal's MST, detecting redundant connections, percolation theory.
  • →
    Topological sort (Kahn's BFS)Count in-degrees. Start with all zero-in-degree nodes. Process: output, then decrement neighbors' in-degrees; enqueue those that reach zero. Cycle exists iff output count < V.
  • →
    Dijkstra's algorithmGreedy: always relax the closest unvisited node (min-heap priority queue). Invariant: once extracted, a node's distance is final (no negative edges allowed to violate this). Backbone of GPS routing, network path selection.
  • →
    Bipartite check / graph coloringBFS/DFS with 2-coloring. Assign alternating colors to neighbors. If you ever need to assign both colors to a node, the graph is not bipartite. Used in: matching problems, scheduling, conflict detection.
🗺️
Maps & Navigation
Cities are nodes; roads are edges with weights (distance/time). Dijkstra/A* finds optimal routes in Google Maps.
👥
Social Networks
Users = nodes; friendships = edges. BFS finds degrees of separation. Community detection finds clusters.
📦
Package Managers
npm/pip dependency graphs are DAGs. Topological sort determines install order. Circular dependency = cycle in DAG.
🌐
The Internet
Routers and links form a weighted graph. BGP protocol finds optimal paths across the global internet graph.
🧠
Neural Networks
Neurons are nodes; synapses are weighted directed edges. The whole brain is a massive graph. Inference = forward pass on a DAG.
🧬
Knowledge Graphs
Google's Knowledge Graph: entities are nodes, relationships are edges. Enables semantic search and reasoning.

Algorithm Patterns Atlas

The moves that cut across all structures
First Principle
Patterns are not structure-specific — they are reasoning templates that apply wherever their preconditions hold. Two Pointers works wherever there is linear order. Binary Search works wherever there is monotonicity. Dynamic Programming works wherever there are overlapping subproblems. Recognising the pattern is the first half of solving any problem.
Algorithm Patterns
  • → Two Pointers — sorted/converging linear structure
  • → Sliding Window — contiguous sub-array with constraint
  • → Binary Search — monotone function or sorted array
  • → Bit Manipulation — integer as fixed-width binary
  • → Prefix Sums — range aggregates from precomputed array
  • → Monotonic Stack/Queue — maintain sorted invariant in stack
  • → Fast & Slow Pointers — cycle detection, midpoint
Algorithmic Paradigms
  • → Greedy — local optimal + greedy choice property
  • → Divide & Conquer — split, solve, merge (MergeSort, QuickSort)
  • → Dynamic Programming — overlapping subproblems + optimal substructure
  • → Backtracking — DFS with pruning, build-undo state
  • → Graph BFS/DFS — systematic exploration with visited set
  • → Union-Find — near-O(1) dynamic connectivity

    2D / Knapsack DPState = (index, remaining_capacity). dp[i][w] = best value using first i items with weight limit w. Covers: 0/1 knapsack, partition equal subset sum, target sum.
  • →
    Interval DPState = (l, r) range. dp[l][r] depends on dp[l][k] and dp[k+1][r] for all k in [l,r]. Fill by increasing interval length. Examples: matrix chain multiplication, palindromic substrings (Manacher is better), burst balloons.
  • →
    Subsequence DP (LCS, Edit Distance)State = (i,j) = prefixes of both strings. dp[i][j] depends on dp[i-1][j], dp[i][j-1], dp[i-1][j-1]. Covers: LCS, edit distance (Levenshtein), shortest common supersequence, regex matching.
  • →
    Tree DPCompute dp values bottom-up in post-order. Each node combines children's values. Examples: diameter, max path sum, number of nodes in largest BST, tree independence number.
  • →
    Bitmask DPState includes a bitmask representing subset of elements used. dp[mask][last] = optimal for visiting exactly the nodes in mask, ending at last. Used for: TSP, minimum cost to visit all nodes, assignment problems with n ≤ 20.
