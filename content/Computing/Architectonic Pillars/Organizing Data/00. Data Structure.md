### 1. Arrays & Matrices

**First Principle**: Contiguous block of memory with O(1) random access by index. **Invariants**: Fixed size (or dynamic resize), elements of uniform type, index arithmetic is direct memory offset. **Derived Patterns** (from roadmap’s 11 Array/Matrix subpatterns + Two Pointers/Sliding Window/Binary Search):
- Traversal & aggregation (prefix sums, frequency maps) – exploit direct index math.
- Two-pointer techniques (converging, fast-slow, fixed-gap) – maintain two indices whose relative movement preserves ordering or sum invariants.
- Sliding window (fixed/variable size with constraint/frequency map) – window boundaries move while maintaining the invariant that the sub-array satisfies a condition.
- Partitioning & reordering (Dutch flag, move zeros) – in-place swaps preserve contiguous layout.
- Binary search variants (on value, on answer, rotated array, first/last occurrence) – monotonicity invariant allows logarithmic halving.
- Prefix/suffix tricks (difference arrays, prefix XOR) – precompute aggregates to answer range queries in O(1).

### 2. Strings

**First Principle**: Immutable (in most languages) contiguous character array with O(1) access. **Invariants**: Length fixed at creation; character encoding consistent. **Derived Patterns** (6 subpatterns + overlaps):

- Two-pointer palindrome/anagram checks.
- Sliding window for longest substring without repeating characters.
- HashMap frequency or rolling-hash for pattern matching.
- Prefix/suffix for longest common prefix or KMP-style preprocessing.

### 3. Linked Lists

**First Principle**: Nodes with value + pointer to next (and optionally prev); non-contiguous memory. **Invariants**: Each node reachable only via pointers; no direct index access. **Derived Patterns** (4 subpatterns + Two Pointers):

- Pointer-walk (find middle, cycle detection via Floyd’s tortoise-hare).
- In-place reversal (whole list, sub-list, k-groups) – reverse pointer directions while preserving reachability.
- Merge & sort (merge two sorted lists, merge k lists via heap).
- Structural transforms (partition around pivot, reorder odd/even).

### 4. HashMap / HashSet

**First Principle**: Array of buckets + hash function mapping key → bucket index; average O(1) access. **Invariants**: Hash collision resolution (chaining or open addressing); load factor controls resizing. **Derived Patterns**:

- Counting & frequency (anagrams, majority element, top-k).
- Existence/complement lookup (two-sum, de-duplication).
- Mapping relations (parent-child for graphs, index-to-position).
- Caching/memoization tables.

### 5. Stack & Queue

**First Principle**: LIFO (stack) or FIFO (queue) with O(1) push/pop at ends. **Invariants**: Order of insertion determines removal order. **Derived Patterns** (6 Stack subpatterns):

- Parsing (parentheses, path canonicalisation).
- Monotonic stack (next greater, stock span, largest rectangle).
- State simulation (undo/redo, backtracking).
- Queue for BFS (level-order, shortest unweighted path).

### 6. Binary Search (applicable to sorted arrays/BSTs)

**First Principle**: Sorted order invariant allows logarithmic elimination of half the search space. **Derived Patterns** (5 subpatterns):

- Standard search, first/last occurrence.
- Parametric search (binary search on answer).
- Rotated sorted array search.
- Search in 2D matrix.

### 7. Sliding Window (array/string specific)

**First Principle**: Maintain a dynamic contiguous sub-array while moving boundaries in one direction only. **Derived Patterns** (4 subpatterns):

- Fixed-size window.
- Variable-size with constraint.
- Minimum/maximum window.
- Window with frequency map.

### 8. Bit Manipulation

**First Principle**: Integers as fixed-width binary representation; bitwise operations are O(1) on word size. **Derived Patterns** (4 subpatterns):

- XOR cancellation (single number, missing duplicate).
- Bit counting & masks (Hamming weight, subset enumeration).
- Power-of-two checks and parity tricks.

### 9. Heaps / Priority Queues

**First Principle**: Complete binary tree satisfying heap-order property (parent ≥/≤ children). **Invariants**: Root is always min/max; O(log n) insert/extract. **Derived Patterns** (4 subpatterns):

- Top-k selection & frequency-based.
- Scheduling & greedy (interval, task with cooldown).
- Streaming (running median via two heaps).
- K-way merge.

### 10. Trees (Binary, BST, Generic)

**First Principle**: Hierarchical nodes with parent-child pointers; no cycles. **Invariants**: Unique paths from root; BST maintains ordered sub-tree property. **Derived Patterns** (7 Tree Traversal subpatterns):

- Traversal families (pre/in/post-order recursive/iterative/Morris, level-order, zigzag).
- Root-to-leaf paths (path sum, serialization).
- Structural properties (height, diameter, LCA, subtree checks).
- BST-specific (kth smallest, floor/ceil, in-order iterator).

### 11. Graphs

**First Principle**: Nodes + edges (adjacency list/matrix); may contain cycles. **Invariants**: Reachability via edges; undirected edges are bidirectional. **Derived Patterns** (7 Graph Traversal subpatterns):

- Traversal templates (BFS single/multi-source, 0-1 BFS, iterative DFS).
- Connectivity & components (Union-Find with path compression/union-by-rank, bipartite check).
- Cycle detection (topo sort + Kahn’s, DFS colour).
- Shortest paths (BFS unweighted, Dijkstra, Bellman-Ford).
- Topological sort & dependencies.
- Strongly Connected Components (Kosaraju/Tarjan) & bridges/articulation points.
- Tree-on-graph (DFS tree, LCA on trees treated as graphs).

### 12. Greedy Algorithms

**First Principle**: Locally optimal choice leads to globally optimal solution when problem exhibits optimal substructure + greedy choice property. **Derived Patterns** (5 subpatterns):

- Interval scheduling, activity selection.
- Fractional knapsack, Huffman coding.
- Minimum spanning tree (Kruskal/Prim).

### 13. Backtracking

**First Principle**: Depth-first exploration with pruning; state is built incrementally and undone on failure. **Derived Patterns** (7 subpatterns):

- Subset/permutation/combination generation.
- Sudoku/N-Queens style constraint satisfaction.
- Word search, maze traversal.

### 14. Dynamic Programming

**First Principle**: Optimal solution to problem = optimal solutions to overlapping subproblems stored in table (memoisation or tabulation). **Derived Patterns** (11 subpatterns):

- Linear DP (knapsack, LIS, partition).
- Interval DP (palindromic substrings, matrix-chain).
- Subsequence/subset DP (LCS, edit distance, subset sum).
- Tree DP (subtree sums, independent sets).
- Bitmask DP (TSP, subset optimisation).

### 15. Segment Tree & Fenwick Tree (Binary Indexed Tree)

**First Principle**: Tree over array that stores aggregate information on ranges; enables O(log n) range queries and point updates. **Derived Patterns** (2 subpatterns):

- Range sum/min/max queries.
- Lazy propagation for range updates.

### 16. Design Patterns (LLD context)

**First Principle**: Separation of concerns + encapsulation; interfaces define contracts. **Derived Patterns**:

- Cache designs (LRU, LFU).
- Randomized structures (O(1) insert-delete-getRandom).
- Ordered structures (min/max stack, monotonic queue).
- Trie for prefix queries.

- **Abstract Data Type (ADT):** interface, operations, invariants.
    
- **Complexity analysis:** worst/average/best-case time and space; amortized cost.
    
- **Memory layout:** contiguous (arrays) vs pointer-based (linked structures), cache effects.
    
- **Mutability & persistence:** destructive updates vs immutable/persistent variants.
    
- **Trade-offs:** time vs space, simplicity vs generality, worst-case vs average-case.
    
- **Correctness techniques:** invariants, pre/post-conditions, loop invariants, formal proofs.
    
- **Concurrency basics:** thread-safety, locks, lock-free primitives (high-level awareness).
    
- **API design & testing:** clear contracts, edge-case tests, property-based and unit tests.

  
If input array is sorted then  
- Binary search  
- Two pointers  
  
If asked for all permutations/subsets then  
- Backtracking  
  
If given a tree then  
- DFS  
- BFS  
  
If given a graph then  
- DFS  
- BFS  
  
If given a linked list then  
- Two pointers  
  
If recursion is banned then  
- Stack  
  
If must solve in-place then  
- Swap corresponding values  
- Store one or more different values in the same pointer  
  
If asked for maximum/minimum subarray/subset/options then  
- Dynamic programming  
- Sliding window  
  
If asked for top/least K items then  
- Heap  
- QuickSelect  
  
If asked for common strings then  
- Map  
- Trie  
  
Else  
- Map/Set for O(1) time & O(n) space  
- Sort input for O(nlogn) time and O(1) space

**Concepts to Learn**
1. SOLID Principles  
    The 5 principles of object-orientated programming create clean, maintainable and scalable code.  
2. Domain-Driven Design  
   Improve your understanding of complex business domains and learn to model them using DDD principles.  
3. Clean Architecture  
   Structure your code in a way that enforces separation of concerns, promotes testability and ensures your application remains flexible.  
4. Test-Driven Development  
   Write more reliable and maintainable code. Write tests before you write the code to help ensure your code meets the required specifications and reduces the likelihood of bugs.  
5. Continuous Integration & Continuous Delivery  
  Streamline your software development process, enabling faster delivery through automated builds, tests and deployment.
